# 🚴 AdventureWorks Sales Analytics — Power BI

An end-to-end Business Intelligence project built for AdventureWorks — a global retail company — using Microsoft Power BI. This intermediate-level project covers the complete BI workflow: multi-source data import, Power Query transformation, star schema modelling, advanced DAX measures, and a 3-page interactive dashboard designed for executive decision-making.

---

## ✨ Key Features

- **Executive Sales Dashboard** — 4 KPI cards (Revenue LY: $18M, Total Orders: 15K, Total Profit: $9M, Profit Margin: 32.71%), Monthly Revenue Trend comparing current vs last year, Revenue by Category bar chart, Revenue by Country filled map, Revenue by Order Size donut chart, and interactive slicers for Year, Quarter, Product Category, and Country
- **Customer & Product Deep Dive** — Top 10 Products by Revenue table, Product Analysis scatter chart (Revenue vs Profit Margin %), Revenue by Age Group bar chart, Revenue vs Orders by Month dual-axis chart, Revenue by Category × Quarter matrix, and a dynamic Top Performing Region card (Australia leads!)
- **Business Insights & Recommendations** — Key Findings text box, Decomposition Tree breaking Total Revenue by Category → Region → Age Group, Key Influencers visual analysing what drives High order size, and actionable business recommendations
- **Star Schema Data Model** — Sales_Data as the central fact table connected to 6 dimension tables with proper Many-to-One relationships
- **10+ DAX Measures** — All stored in a dedicated `_Measures` table including Time Intelligence (YTD, LY, YoY Growth) and profitability KPIs



## 🔍 Key Insights from the Data

- **Australia is the top performing region** — generating the highest revenue across all territories, confirmed by both the map visual and the Decomposition Tree
- **Bikes category dominates** — contributing $22.6M+ out of total revenue, far ahead of Components ($3.2M), Clothing ($0.4M), and Accessories ($0.2M)
- **September is the peak revenue month** — hitting $2.43M, indicating strong seasonal demand
- **High order size is strongly linked to Bikes** — the Key Influencers visual shows Bikes category increases the likelihood of a High order size by 17.7%
- **Road-150 Red 62 is the top product** — generating $55.49M in revenue with a 31.43% profit margin
- **Senior age group leads revenue** at $13M, followed by Mid-Career at $11M and Young Adults at $3M
- **Profit Margin held at 32.71%** across the full dataset

## 📁 Data Model — Tables

| Table | Type | Description |
|---|---|---|
| `Sales_Data` | Fact Table | Transaction-level sales records with revenue & order size |
| `AdventureWorks_Customers` | Dimension | Customer demographics, Full_Name, Age_Group |
| `AdventureWorks_Products` | Dimension | Product catalog with pricing and Profit_Margin column |
| `AdventureWorks_Product_Categories` | Dimension | High-level product groupings |
| `AdventureWorks_Product_Subcategories` | Dimension | Mid-level product groupings |
| `AdventureWorks_Territories` | Dimension | Sales territory and region data |
| `AdventureWorks_Calendar` | Date Table | Date reference with Year, Month, Quarter, Fiscal_Quarter, Day of Week |
| `_Measures` | Measures Table | All DAX measures organized in one place |

---

## 📐 DAX Measures Built

| Measure | Business Purpose |
|---|---|
| `Total Revenue` | Sum of all sales revenue |
| `Total Orders` | Count of all transactions |
| `Total Customers` | Distinct count of customers who purchased |
| `Avg Order Value` | Revenue per transaction on average |
| `Total Profit` | Net profit using SUMX with RELATED product cost |
| `Profit Margin %` | Profitability as a percentage |
| `Revenue YTD` | Revenue accumulated from start of year using TOTALYTD |
| `Revenue LY` | Same period last year using SAMEPERIODLASTYEAR |
| `YoY Revenue Growth %` | Year-over-year growth percentage |
| `Top Product Revenue` | Revenue generated by the single top product using MAXX + TOPN |

## 🗺️ Data Model Relationships

| From Table | To Table | Key | Cardinality |
|---|---|---|---|
| Sales_Data | Customers | CustomerKey | Many-to-One |
| Sales_Data | Products | ProductKey | Many-to-One |
| Sales_Data | Territories | TerritoryKey | Many-to-One |
| Sales_Data | Calendar | OrderDate → Date | Many-to-One |
| Products | Product_Subcategories | ProductSubcategoryKey | Many-to-One |
| Product_Subcategories | Product_Categories | ProductCategoryKey | Many-to-One |
